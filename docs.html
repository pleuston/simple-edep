simple-edep

A static, client-side editor for encoding Roman/Latin inscriptions as EpiDoc/TEI XML and committing them to GitHub. There is no backend: the XML is built in your browser and saved through the GitHub Contents API with your own personal access token.

Signing in & saving

You can browse the catalogue, map and reading views without an account. To save, create a GitHub personal access token with public_repo scope (or repo for a private data repo) and paste it on the sign-in page. The token lives only in this browser. “② Save to GitHub” commits the record to records/<file>.xml; you can also Copy or Download the XML and commit it by hand.

The Leiden+ editor

The transcription pane is a live Leiden+ → EpiDoc editor (jinn-codemirror). Type with Leiden conventions and the EpiDoc XML is produced automatically. You can also paste raw text from EDCS or PHI and use the Import buttons to convert it.

Use the toolbar buttons for the common cases, and ↩ to (re)number lines.

What gets encoded

  • Identity & IDs — title, editor, and cross-references (EDH, EDCS, EDR, Trismegistos, PHI, CIL → idno).
  • Object & support — object type and material (with EAGLE / Getty AAT URIs), dimensions, letter heights, condition.
  • Layout & lettering — columns, lines, script (handNote).
  • Text type & dating — EDH text type, origDate with @when/@notBefore/@notAfter/@datingMethod, place (Pleiades URI), province.
  • Find spot — modern country/region/settlement, coordinates (for the map), provenance, repository.
  • Edition, apparatus, translation, commentary, bibliography — standard EpiDoc div structure.

Every record validates as well-formed XML against the EpiDoc RNG processing instructions and follows the EpiDoc Guidelines.

Data & deployment

Records live in records/*.xml in the public repository pleuston/simple-edep, served as static files. The catalogue and map read data/records-index.json; rebuild it after bulk-editing with node scripts/build-index.js.
